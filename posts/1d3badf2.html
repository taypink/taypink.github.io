一、概述&核心机制

  • 128bit，HEX
  • 取消Broadcast，使用Multicast/Anycast
  • 固定头部为40B瘦身，引入Extension Header(如ESP、AH)
  • 不能实现MAC过滤（L3协议）

报文结构：

​

  • 基本头部(Base)：取消了v4的Header Length、Identification、Flags、Fragment Offset、Header Checksum

  • 拓展头部(Extension)：分片、路由、安全 AH/ESP

IPv6 地址分类

  • 固定Multicast地址：

    FF02::1（所有节点）、FF02::2（所有路由器）

    FF02::5 / ::6（OSPFv3）

    FF02::16（被请求节点组播地址，用于替代 ARP）

  • 常见缩写：

    GUA→全球单播、LLA→链路本地、ULA→唯一本地


LLA

Link-Local Address(链路本地地址)

  • 生成方式：

    自动─EUI-64/软件

    手动配置（优先级最高）

GUA

Global Unicast Address(全球单播地址)

  • 每个接口可以有10个网络前缀不同的全球单播地址。

NDP

  • 基于ICMPv6(Type 133-137)，= IPv4 状态机+ARP

  • 使用Multicast，视作L3协议→可以使用L3的安全协议

Neighbor Discovery Protocol，邻居地址发现协议，替代了v4的ARP

机制和报文对应关系

机制  ICMPv6 报文 作用 RS(133) RA(134) NS(135) NA(136)
AR→Address Resolution
(地址解析)		 组播 NS 索要对方的 MAC 地址，对方用 NA单播回应 ✓ ✓
RD→Router Discovery
(路由器发现/前缀公告)		 终端UP后主动发RS找网关，路由器定期发/响应RA ✓ ✓
SLAAC→Stateless Address Autoconfiguration
(无状态地址自动配置)		 终端结合 RA 报文中的前缀，自己计算并生成 IPv6 地址（即插即用、EUI-64）。 ✓ ✓
DAD
(重复地址检测)		 使用新地址前先发 NS 探路，没收到NA回应，就证明冲突，接口熔断进入Duplicate状态 ✓ ✓
NUD→Neighbor Unreachability Detection<br />(邻居状态跟踪) 单播NA探测邻居是否存活
维护一张邻居高速缓存表，指导状态机在 5种状态之间进行精细切换		 ✓
PR→Prefix Renumbering
(前缀重新编址)		 当运营商更换前缀时，路由器通过 RA 报文通告新前缀并使旧前缀老化，实现全网平滑更换 IP。 ✓ ✓
Redirect
(路由重定向)		 网关发现有更优的下一跳路径时，发此报文告知源端设备优化路由 重定向 (137)
✓

报文类型

Used For？
RS
(Router Solicitation)-133		 主机请求RA
目的地址为所有Router的组播地址（ff02::2）
NS
(Neighbor Solicitation)-135		 探测对方 MAC
目的 IP 为对方的Solicited-node Addr (Multicast)
类似ARP Request报文
RA
(Advertisement)-134		 通告网络前缀与参数，定期发送或回应 RS
目的 IP 为所有节点的组播地址（ff02::1）或主机单播地址
NA
(Advertisement)-136		 回应自身 MAC
类似ARP Reply报文
Redirect-137

机制详解

AR-Address Resolution

替代IPv4的ARP

DAD

用于探测地址是否使用，Duplicate Address Detect

  • 使用任意Unicast地址都需要先DAD，包括LLA
  • 还未DAD的地址称为 Tentative Address(试验地址) ，此时该接口不能使用这个地址
  • 若2个节点配置同一个地址，DAD时，一方收到对方的 DAD NS 报文，则接受方无法使用该地址

NUD

Neighbor Cache State(邻居状态机)

\[\text{Incomplete} \rightarrow \text{Reachable} \rightarrow \text{Stale} \rightarrow \text{Delay} \rightarrow \text{Probe}\]

五种状态的转换条件（如：收到非请求的 NA 会导致状态退化到 Stale；在 Delay 状态下定时器超时会进入 Probe 发送单播 NS 进行探测）。

路由协议适应性（以 OSPFv3 为主）

对比维度 OSPFv2 (IPv4) OSPFv3 (IPv6)
运行基础 基于 IP 子网（Subnet）运行。 基于 链路（Link）运行，拓扑与 IP 编址完全解耦。
邻居建立 使用接口的公网/私网 IPv4 地址。 强制使用 LLA（FE80::/10） 建立邻居，哪怕接口没有配置 GUA。
LSA 传递 1类、2类 LSA 既传拓扑，又传路由信息。 1类、2类 LSA 只传拓扑；路由信息剥离到 8类（Link-LSA）和 9类（Intra-Area-Prefix-LSA）中传递。
多实例支持 不支持，单接口只能运行一个实例。 支持 Instance ID 字段，单链路上可运行多个独立实例。
