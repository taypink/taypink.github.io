IS-IS Protocol

发表于2026-01-03|更新于2026-06-07|NetworkR&S
|字数总计:1.1k|阅读时长:4分钟|阅读量:

IS-IS

Level 级别的单向隔离：

  • 单向流动：Level-1 的单向路由信息可以被 L1-2 路由器注入到 Level-2 中。
  • 但是，Level-2 的拓扑 LSP 绝对不会倒流进 Level-1 的 LSDB 中。

NET地址

  • 每台运行IS-IS的网络设备至少需拥有一个NET，一台设备可以配置多个NET，但是 NET的SystemID必须相同。
  • SystemID的长度一般是6Byte，一般根据RouterID配置SystemID。

Area ID

标识设备所在的区域

A级 IP 很高（High）/AIH

\[\text{Area ID} = \text{AFI} + \text{IDI} + \text{High Order DSP}\]

AFI

机构与格式标识符，用于指定地址分配机构 and 格式

(Authority and Format Identifier)

49.0001里的49

IDI

初始域标识符，标识具体的网络

(Initial Domain Identifier)

High Order DSP

高位域特定部分，在域内进行更高级、更精细的区域划分

(High Order Domain Specific Part)

System ID

对应49.0001.0000.0000.0001.00 斜体下划线部分

3 段： 6 字节 = 48 位 = 12 个十六进制数

只有广播网络（Ethernet）才有伪节点，点对点（P2P）网络没有伪节点

SEL

服务选择符。在 IP 网络中运行 IS-IS 时，必须固定为 00。

\[\underbrace{49}_{\text{AFI}} . \underbrace{0001}_{\text{IDI + High Order DSP}} . \underbrace{0000.0000.0001}_{\text{System ID (6字节)}} . \underbrace{00}_{\text{SEL (1字节)}}\]

LSP交互产生原因：

疑点突破

在IS-IS广播网中，同一网段上的同一级别的路由器之间都会形成邻接关系，包括所有的非DIS路由器之间全网状（Full Mesh）的邻接关系。

正确，核心原因：DIS 没有“备份”，不承担中转职责

  • 在IS-IS 没有“备份 DIS”的概念（没有类似 BDR 的角色）。当有新的LSP报文产生时，他不会单独发给DIS，而是通过特有的组播地址泛洪出去。无需像OSPF那般，依赖DIS做中转，因为都是邻接的状态。

    (Level-1 →01-80-C2-00-00-14，Level-2 →01-80-C2-00-00-15)

Router-id 转化为System ID，以10.10.1.1为例

外部路由引入到IS-IS中时，默认等级为Level-2

正确,因为只有放在 Level-2，才能保证全网可达；如果放进 Level-1，路由就被孤立在某个小区域里了

这是由ISIS的防环机制决定的,Level-1 区域的路由会通告给 Level-2（骨干），但 Level-2 的路由默认绝不会通告给 Level-1.
