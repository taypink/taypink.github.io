一、概述&核心机制

  • TCP 179

  • 触发更新(增量)

  • 运行在AS之间，只传递路由信息，不计算路由，不暴露AS内部拓扑

水平防环

IBGP-Split Horizon

从其他 IBGP peer学到的路由，不会再转发给其他的 IBGP peer

若要打破，需靠 RR（结合 Originator_ID / Cluster_List）或联盟

EBGP-AS_Path

只要收到含有自己 AS 号的路由，一律丢弃

路由引入与通告

  • BGP不生产路由，只搬运路由
  • 必须network/import IGP路由进入BGP数据库

引入

范围 特征
network 1.本地路由表必须存在
2.引入的路由掩码必须和本地路由表的一致		 精确手术刀
适合宣告核心业务网段
redistribute/import 引入IGP路由、直连、静态路由 大水漫灌，但容易引入垃圾路由或引发环路
必须强制关联 「route-policy」 进行过滤与属性修饰

通告

三大核心铁律（指导 BGP 路由传递的底层逻辑）:Only-Best、EBGP 、Spilt Horizon

  • 只传最优：本地 BGP 表中只有获得 *> 的优选路由才会通告给邻居
  • EBGP 传全网：从 EBGP 邻居学到的最优路由，可以传给其他所有的 EBGP 和 IBGP 邻居
  • IBGP 水平分割（防环死锁）：从 IBGP 邻居学到的路由，绝不传给另一个 IBGP 邻居。由此引发 Full-Mesh 大规模组网痛点，后续衍生出 RR（非非不传）和联盟。

路由聚合

  • 手动聚合 > 自动聚合 > network > import（华为明确定义）

手动聚合-Aggregate

  • 无论 network 还是 import 引入，只要本地 BGP 路由表里存在的明细路由，均可手动聚合

  • 控制参数：

    cisco 华为
    summary-only detail-suppressed 抑制明细,开启后只通告聚合后的全网段，不通告明细网段（不加此参数则聚合路由和明细路由一起发）。
    as-set as-set 携带 AS 路径, 聚合路由会继承明细路由的 AS_PATH，防止EBGP环路。
    如果不加此参数，聚合路由会丢失明细路由的 AS_PATH，可能引发路由环路。
    suppress-map suppress-policy 选择性抑制。 仅抑制部分明细，放行部分明细

自动聚合-Auto-Summary

  • 仅对import有效，对network不生效

  • 一般不用，颗粒度太大： 强行将明细路由聚合为主类网络（Classful，如 10.1.1.0/24 变成 10.0.0.0/8），并自动抑制（删除）明细路由

路由表处理

$$\substack{\text{Received UPDATE} \ \text{收到邻居更新}} \rightarrow \substack{\text{Adj-RIB-In} \ \text{入方向邻居表}} \rightarrow \substack{\text{Inbound Policy} \ \text{入方向策略过滤}} \rightarrow \substack{\text{Loc-RIB} \ \text{本地 BGP 总表}} \rightarrow \substack{\text{Decision Process} \ \text{十三步选路决策}} \rightarrow \begin{cases} \substack{\text{IP-RIB} \ \text{下发 IP 路由表}} \ \ \substack{\text{Outbound Policy} \ \text{出方向策略过滤}} \rightarrow \substack{\text{Adj-RIB-Out} \ \text{出方向邻居表}} \rightarrow \substack{\text{Sent to Peer} \ \text{发送给邻居}} \end{cases}$$

Adj-RIB─in

  • 接受邻居传来的原始UPDATE报文

Inbound Policy

  • 调用cisco:Route-map/华为：Route-policy、IP-Prefix，AS-PATH filter等过滤邻居传入的路由
  • 过滤后的路由打上本地属性(如Local-Preference)
  • 过滤后路由有状态符号*:路由合法可用

Loc-RIB

  • 无论下一跳是否可达，收到的路由都会先存入这里
  • 不可达路由一样能拿到*

Decision Process

  • 面试第一：经过十三步选路后，最优路由会获得>标识，此时组成*>

IP-RIB

  • 只有通过选路胜出的 Best 路由*>，才会下发到全局 IP 路由表，指导数据转发

Outbound Policy

  • 调用策略，决定可以发送的最佳路由，发送出去是否要修改属性(例如加长 AS-Path 伪造路径)

Adj-RIB-Out

准备发送给特定邻居的报文队列

行为&意义

Connect 等待 TCP 三次握手完成
成功→Opensent&发送 OPEN 报文

• 确认邻居类型：收到邻居的Open报文，确认对方AS号与本地AS号是否一致，一致就IBGP，反之EBGP
• 协商Hold Time：对比各自报文中的Hold Time，强制采用两个中最小的那个作为最终的Hold Time

Error:
1.Hold Timer超时(在协商时间内始终未收到对方的 Keepalive 报文)→回退到Idle
2.收到邻居的Notification→回退到Idle
Established 邻居关系建立完成
通过 Update 报文传递路由信息，并周期性发送 keepalive 维持心跳
收到Notification→回退到Idle

  • 在连接过程中，比较Router-ID，数值大的路由器将保持其主动发起的TCP连接状态，另一台则关闭自己的连接处于被动状态。

UPDATE报文

$$\text{UPDATE} = \text{Withdrawn Routes} + \text{Path Attributes} + \text{NLRI}$$

字段名称 (Field) 中文释义 核心承载内容与作用 Hook/记忆锚点
Withdrawn Routes 可撤销路由 包含因网络拓扑变动而需要从邻居路由表中删除的、不可达的 IP 路由前缀列表。 告诉邻居：“哪些路由死掉了”
Path Attributes 路径属性 描述路由的一系列核心特征和度量（如 AS_Path、Next_Hop、MED 等），用于 BGP 选路决策。 告诉邻居：“这些路由质量如何”
NLRI 可达路由信息 包含新通告的、当前可达的 IP 路由前缀及掩码列表。Network Layer Reachability Information。 告诉邻居：“我又发现了哪些新路”

面试第一：经过十三步选路后，最优路由会获得>标识，此时组成*>

Warning:
获得了 BGP 的 *> 只代表拿到了“推荐信”，能不能最终写进 IP 路由表，还要看AD / Preference（管理距离 / 协议优先级）

  • Cisco：EBGP 的 AD = 20，IBGP = 200。如果此时 OSPF（AD=110）传过来一条一样的明细路由，EBGP（20 < 110）能胜出进入 IP 路由表；而 IBGP（200 > 110）则会在进入 IP 路由表时被 OSPF 击败。
  • 华为：无论是 EBGP / IBGP，BGP 的协议优先级默认统一是 255。
    这意味着在华为设备上，任何 IGP 协议（如 OSPF = 10/150，ISIS = 15）都能在入 IP 路由表时轻松“碾压” BGP。

ECMP/负载均衡

  • BGP 的负载分担完全取决于十三步选路的第 10 步（Cisco原则）
  • 因为BGP 默认“单通路”，无论存在多少条合法路径，最终只有一条最优路由（即标记为 *>）会被注入到 IP 路由表中。要让 BGP 路由在第10步执行「负载分担」，多条路径必须在选路的前9步属性中完全保持一致。

  1. Weight/Preferred-Value相同

  2. Local-Preference相同

  3. 都是聚合路由或者非聚合路由

  4. AS_Path 长度相同(长度要一致，AS 号及顺序也必须完全一致)

    可以打破此限制：cisco：bgp bestpath as-path multipath-relax

    ​ 华为：balance as-path-relax

  5. Origin类型（IGP、EGP、Incomplete）相同

  6. Metric/MED一致

  7. 都是EBGP路由 或 都是IBGP路由

  8. AS内部IGP的Metric相同

五、大规模组网

出现原因：
因为BGP 的水平分割机制规定“从 IBGP 学习到的路由不能传给其他 IBGP 对等体”。
这导致大规模组网下必须建立Full-Mesh(全互联)，其连接数呈 $n(n-1)/2$ 几何级数暴涨，性能要求高，所以出现了RR和Confederation

RR反射器-Route Reflector

通过划分“特权设备”来打破水平分割。

Client 之间不需要建立任何邻居关系，全靠 RR 居中“传话”

RR会将学习的路由反射出去，从而IBGP路由在AS内传播无需建立IBGP全互联。

将一台BGP路由器指定为RR时，还需要指定其Client。至于Client本身，无需做任何配置，它并不知晓网络中存在RR

核心规则

非非不传：从非客户学到的不会传给非客户，其他情况都传(只在AS内部传递)

  • 从 Client 学到的路由： RR（Client） → Client 和 Non-Client。
  • 从 Non-Client 学到的路由：RR（Non-Client） → Client（Non-Client 之间依然被水平分割）。
  • 从 EBGP 学到的路由： RR （EBGP）→ Client 和 Non-Client

RR防环

RR会使得IBGP水平分割原则失效，可能导致环路的产生，所以RR会为BGP路由添加两个特殊的路径属性来避免出现环路：
Originator_ID、Cluster_List；这两都属于Optional Non-transitive

Originator ID
  • 由RR产生，RR反射路由出去时会添加该属性，值为通告该路由的Router的Router-id
  • 如果AS内有多个RR，则Originator_ID由第一个RR创建，并且不会被后续的RR更改
  • 与AS_Path类似，当Router收到的路由的Originator_ID = 自己的Router ID时，会忽略该条路由更新


Originator ID传递

Ref:BGP路由反射器、路由优选-最爱喝白开水

Cluster_List

  • 每一个簇都有唯一的Cluster_List，默认为RR的BGP Router-id

  • 当 RR 收到一条路由并反射出去时，它会在路由的属性中加入自己的 Cluster_ID

    如果 RR 收到一条路由，发现列表里已经有自己的 Cluster_ID 了，说明发生了内部环路，直接丢弃。

联盟-Confederation

[对外一体，对内自治]：在大AS外部看来，整体是一个AS（对外通告的AS叫做Confederation ID）；但在内部被拆分为多个私有的AS(通常使用64512-65535)

内部关系

  • 子AS间：IBGP
  • 多个子AS间：联盟EBGP
  • 属性传递：联盟 EBGP 邻居在传递路由时，Next_Hop、Local-Preference、MED 等属性默认不丢失，可以直接跨子 AS 传递（这与普通 EBGP 完全不同）。

防环机制

AS_Confed_Sequence和AS_Confed_Set

​ 有序子 AS 列表 无序 集合

[剥离时机]：当路由最终被发送给大 AS 外部真正的 EBGP 邻居时，联盟路由器会自动将这些私有子 AS 属性剥离，外部网络完全感知不到联盟的存在。

RR&Confeder对比

对比维度 RR(路由反射器) Confederation(联盟)
拓扑 不需要更改现有拓扑，兼容性好 需要改变逻辑拓扑
配置 配置方便，仅需配置反射器，客户端无需感知 所有设备需要重新进行配置
连接 集群与集群之间仍需要全连接 联盟子 AS 之间是特殊的 EBGP 连接，不需要全连接
场景 适用于中、大规模网络 适用于大规模网络
