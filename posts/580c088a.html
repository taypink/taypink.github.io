一、概述&核心机制

  • TCP 179
  • 触发更新(增量)
  • 运行在AS之间，只传递路由信息，不计算路由，不暴露AS内部拓扑

水平防环

路由引入与通告

  • BGP不生产路由，只搬运路由
  • 必须network/import IGP路由进入BGP数据库

引入

通告

路由聚合

  • 手动聚合 > 自动聚合 > network > import（华为明确定义）

手动聚合-Aggregate

  • 无论 network 还是 import 引入，只要本地 BGP 路由表里存在的明细路由，均可手动聚合

  • 控制参数：

    cisco 华为
    summary-only detail-suppressed 抑制明细,开启后只通告聚合后的全网段，不通告明细网段（不加此参数则聚合路由和明细路由一起发）。
    as-set as-set 携带 AS 路径, 聚合路由会继承明细路由的 AS_PATH，防止EBGP环路。
    如果不加此参数，聚合路由会丢失明细路由的 AS_PATH，可能引发路由环路。
    suppress-map suppress-policy 选择性抑制。 仅抑制部分明细，放行部分明细

自动聚合-Auto-Summary

  • 仅对import有效，对network不生效

  • 一般不用，颗粒度太大： 强行将明细路由聚合为主类网络（Classful，如 10.1.1.0/24 变成 10.0.0.0/8），并自动抑制（删除）明细路由

路由表处理

\[\substack{\text{Received UPDATE} \\ \text{收到邻居更新}} \rightarrow \substack{\text{Adj-RIB-In} \\ \text{入方向邻居表}} \rightarrow \substack{\text{Inbound Policy} \\ \text{入方向策略过滤}} \rightarrow \substack{\text{Loc-RIB} \\ \text{本地 BGP 总表}} \rightarrow \substack{\text{Decision Process} \\ \text{十三步选路决策}} \rightarrow \begin{cases} \substack{\text{IP-RIB} \\ \text{下发 IP 路由表}} \\ \\ \substack{\text{Outbound Policy} \\ \text{出方向策略过滤}} \rightarrow \substack{\text{Adj-RIB-Out} \\ \text{出方向邻居表}} \rightarrow \substack{\text{Sent to Peer} \\ \text{发送给邻居}} \end{cases}\]

Adj-RIB─in

  • 接受邻居传来的原始UPDATE报文

Inbound Policy

  • 调用cisco:Route-map/华为：Route-policy、IP-Prefix，AS-PATH filter等过滤邻居传入的路由
  • 过滤后的路由打上本地属性(如Local-Preference)
  • 过滤后路由有状态符号*:路由合法可用

Loc-RIB

  • 无论下一跳是否可达，收到的路由都会先存入这里
  • 不可达路由一样能拿到*

Decision Process

  • 面试第一：经过十三步选路后，最优路由会获得>标识，此时组成*>

    Warning: 获得了 BGP 的 *> 只代表拿到了“推荐信”，能不能最终写进 IP 路由表，还要看AD / Preference（管理距离 / 协议优先级）

    • Cisco：EBGP 的 AD = 20，IBGP = 200。如果此时 OSPF（AD=110）传过来一条一样的明细路由，EBGP（20 < 110）能胜出进入 IP 路由表；而 IBGP（200 > 110）则会在进入 IP 路由表时被 OSPF 击败。
    • 华为：无论是 EBGP / IBGP，BGP 的协议优先级默认统一是 255。 这意味着在华为设备上，任何 IGP 协议（如 OSPF = 10/150，ISIS = 15）都能在入 IP 路由表时轻松“碾压” BGP。
IP-RIB
  • 只有通过选路胜出的 Best 路由*>，才会下发到全局 IP 路由表，指导数据转发
Outbound Policy
  • 调用策略，决定可以发送的最佳路由，发送出去是否要修改属性(例如加长 AS-Path 伪造路径)
Adj-RIB-Out

准备发送给特定邻居的报文队列

二、状态机&报文&邻居

六种状态机

报文类型

Message Type Type Code 发送时机与触发条件 核心承载内容 核心作用
OPEN 1 TCP 连接建立成功后发送的第一个报文 BGP版本、本端AS号、Hold Time、Router ID、可选参数（如MP-BGP） 协商参数，建立邻居关系
UPDATE 2 网络拓扑发生变化或路由失效时（触发式） 撤销路由前缀、路径属性（Path Attributes）、可达路由前缀（NLRI） 通告、更新或撤销路由
NOTIFICATION 3 运行或协商过程中检测到任何错误时 错误代码（Error Code）、错误子代码、错误数据 报告错误并立即断开连接
KEEPALIVE 4 邻居建立时确认，以及后续周期性发送（默认60s,age-180s） 无数据正文（仅19字节头部） 维持邻居关系（心跳机制）
ROUTE-REFRESH 5 本地路由策略改变，需要重新过滤路由时（触发式） 地址族标识符（AFI/SAFI） 动态请求邻居重发路由（不中断连接）

报文交互流程


BGP 报文交互

UPDATE报文

\[\text{UPDATE} = \text{Withdrawn Routes} + \text{Path Attributes} + \text{NLRI}\]

字段名称 (Field) 中文释义 核心承载内容与作用 Hook/记忆锚点
Withdrawn Routes 可撤销路由 包含因网络拓扑变动而需要从邻居路由表中删除的、不可达的 IP 路由前缀列表。 告诉邻居：“哪些路由死掉了”
Path Attributes 路径属性 描述路由的一系列核心特征和度量（如 AS_Path、Next_Hop、MED 等），用于 BGP 选路决策。 告诉邻居：“这些路由质量如何”
NLRI 可达路由信息 包含新通告的、当前可达的 IP 路由前缀及掩码列表。Network Layer Reachability Information。 告诉邻居：“我又发现了哪些新路”

1 下一跳不可达的路由直接丢弃 下一跳不可达的路由直接丢弃 前置条件：下一跳（Next_Hop）不可达则直接淘汰。
2 Weight（权重）最高 协议首选值（Preferred-Value）最高 均为本地私有属性，仅在本地有效，不传递。数值越大越好。
3 Local-Preference（本地优先级）最高 Local-Preference（本地优先级）最高 AS 内部传播，控制流量如何流出 AS。数值越大越好。
4 本地发起的路由（next hop = 0.0.0.0） 手动聚合 > 自动聚合 > network > import 本地生成的路由优先级高于从邻居学来的路由。
5 AS_Path 长度最短的路由 AS_Path 长度最短的路由 经过的 AS 数量越少越好（可用于防环与路径评判）。
6 Origin 属性（IGP > EGP > Incomplete） Origin 属性（IGP > EGP > Incomplete） 路由起源类型：i 大于 e 大于 ?。
7 MED 值最低的路由 MED 值最低的路由 告知外部 AS 如何流入本 AS。数值越小越好。
8 EBGP 路由 > IBGP 路由 EBGP 路由 > IBGP 路由 外部邻居（EBGP）传来的路由优先级高于内部邻居（IBGP）。
9 到 BGP 下一跳的 IGP 开销（Cost）最小 到 BGP 下一跳的 IGP 开销（Cost）最小 比较到下一跳地址的内部网关协议（如OSPF）开销，越小越好。
10 存在多条路径时，执行负载分担 存在多条路径时，执行负载分担 如果配置了多路径平衡（Multipath），在此处将流量分摊。
11 外部路由中：最先收到的 EBGP 路由 Cluster_List 长度最短的路由 思科此处锁定最先收到的路由（稳定第一）；华为直接比反射次数。
12 BGP Router ID 最小的路由器发布的路由 Originator_ID 最小的路由 比较路由器的 Router ID 或反射器环境下的起源 ID，越小越好。
13 Cluster_List 长度最短的路由 Router ID 最小的路由器发布的路由 思科在此时才比较反射器路径长度。
